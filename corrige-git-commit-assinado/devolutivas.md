# Guia de Devolutivas - Git Commit Assinado
## Como Escrever uma Devolutiva de Qualidade
### Regras: Tom cordial, profissional, incentivador. PT-BR. Primeiro nome do aluno. Mencionar detalhes especificos. Direto ao ponto. Sem emojis.
### Estrutura: 1. Abertura 2. Corpo (resultado + detalhes) 3. Fechamento (incentivo)
### NAO: copiar mesma para todos, ser vago, listar tudo, ser rude.

## NAO APROVADO
### Exemplo 1
Ola, {Nome}!
Obrigado por enviar a sua resposta ao exercicio!
O commit nao esta assinado. A badge 'Verified' nao foi encontrada no GitHub (campo verified: false).
Configure uma chave GPG em sua maquina, vincule-a ao GitHub, e faca um novo commit assinado com git commit -S.
Reveja esse ponto e, quando finalizar, envie o projeto novamente para correcao.
Bom trabalho!
### Exemplo 2
Ola, {Nome}!
Obrigado por enviar a sua resposta ao exercicio!
O repositorio nao contem um arquivo README.md.
Conforme o enunciado, o repositorio deve conter um README.md (pode ser simples, apenas descrevendo o desafio).
Reveja esse ponto e, quando finalizar, envie o projeto novamente para correcao.
Bom trabalho!
### Exemplo 3

O link enviado nao aponta para um repositorio valido ou acessivel.
Verifique se o repositorio e publico e se o link esta correto.
Reveja esse ponto e, quando finalizar, envie o projeto novamente para correcao.
Bom trabalho!
